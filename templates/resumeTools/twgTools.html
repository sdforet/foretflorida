TWIG

TWIG is the self-described "flexible, fast, secure, template engine for PHP". I am sure that this is a valid description given it's wide acceptance in the Symfony community. I have found that twig is can be as helpful and developer friendly as other templateing engines. The main benefit to using twig is an extensive tag/filter/function/test library that it includes. According to twig, it violates separation of resposibilities to use strtoupper() in PHP so they provide their syntax filter {{ 'welcome'|upper }} to use in the template. They provide a good number of similar features to alter the information passed to the template, some of which border on buisness logic in the template. Of course you don't have to use any of these, and other template engines also include many logical filters in their engines as well.

Using Templates

  1. Templates must be named with double extension ".html.twig"
  2. The base template is evaluated first
  3. Child templates can overwrite the parent's blocks with content of their own (only one level)
  4. If it is not in a block, it will not be replaced
  5. Child templates must declare what they're overwriting by using the extends tag
  6. Variables are not scoped to included templates such as macros that are not in the same file (bad)

Three types of syntax in Twig:

  1. {% ... %} "Do Something": executes statements like conditional logic and for loops
  2. {{ ... }} "Say Something": print this route of variable or expression; filters use pipe
  3. {# ... #} "Comment": results of a comment aren't displayed
@file sample.html.twig

{# loop through the array with values #}
{% for info in array %}
   <h3>{{ info.name }}</h3>
   <p>{{ info.description }}</p>
   {# loop through the innerArray for this info #}
   {% for deepInfo in info.items %}
      {{ deepInfo.name }}
      {% if deepInfo.active == true %}
         <span>Show this text!</span>
      {% endif %}
      <p>{{ deepInfo.description|raw }}</p>
   {% endfor %}
   <hr />
{% endfor %}

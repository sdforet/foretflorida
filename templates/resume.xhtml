Code Philosophy

My goal is to create simple, elegant, & functional user experiences with each project. This extends not only to the user interface, but the code as well. As a gear head, I can appreciate a clean well-maintained car, but I really admire the guy who also keeps the mechanical underpinnings as clean and well-maintained. This same admiration applies to software as well. It is refreshing to crack open the "hood" of well written software and admire the elegant structure of well written code.

In my experience, it is uncommon to find a software developer who empathizes with the user and developer while seeking to create web environments which are both intuitive and functional. While writing code, I make every effort to consider not only the end user, but the other developers who will be maintaining my code. I like the acronym U/dX. Instead of just a great User Experience (UX) I strive for a positive User/Developer Experience (U/dX).
